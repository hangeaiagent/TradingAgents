## 5. 故障排查

### 服务无法启动

### `__pycache__` 缓存导致代码未生效（重要！）

**问题**：`git pull` 更新了 `.py` 文件，但如果文件内容与服务器上已有的相同（例如之前用 rsync 同步过），Git 不会更新文件的修改时间（mtime）。Python 通过比较 `.py` 和 `.pyc` 的 mtime 决定是否重新编译，如果 `.pyc` 比 `.py` 更新，Python 会继续使用旧的 `.pyc` 字节码，导致新代码不生效。

**典型场景**：
1. 先用 rsync 同步代码到服务器（服务器运行了新代码，生成了 `.pyc`）
2. 然后 `git checkout . && git pull` 拉取同样的代码（`.py` 的 mtime 没变或更早）
3. Python 加载的仍是旧 `.pyc`，新代码不生效

**解决方案 — 标准部署流程必须清除 `__pycache__`**：

```bash
ssh -i $HOME/.ssh/id_rsa_google_longterm support@34.92.13.95 "cd /home/support/TradingAgents && git pull origin main && find . -name '__pycache__' -type d -exec rm -rf {} + 2>/dev/null; sudo systemctl restart tradingagents"
```

**规则**：每次部署都必须在 `git pull` 之后、`systemctl restart` 之前清除 `__pycache__`。这是必须步骤，不可省略。

## 6. 部署规则（必须遵守）

> **每次部署到服务器时，必须按以下完整流程执行，不可省略任何步骤：**

```bash
# 完整部署流程（一条命令）
ssh -i $HOME/.ssh/id_rsa_google_longterm support@34.92.13.95 "\
  cd /home/support/TradingAgents && \
  git pull origin main && \
  find . -name '__pycache__' -type d -exec rm -rf {} + 2>/dev/null; \
  sudo systemctl restart tradingagents && \
  sleep 2 && \
  sudo systemctl status tradingagents --no-pager"
```

关键步骤说明：

| 步骤 | 命令 | 不可省略原因 |
|------|------|-------------|
| 1. 拉取代码 | `git pull origin main` | 获取最新代码 |
| 2. 清除缓存 | `find . -name '__pycache__' -exec rm -rf {} +` | 避免旧 `.pyc` 导致新代码不生效 |
| 3. 重启服务 | `sudo systemctl restart tradingagents` | 加载新代码 |
| 4. 验证状态 | `sudo systemctl status tradingagents` | 确认服务正常运行 |

> **禁止**：不要单独使用 rsync 同步代码后不清除 `__pycache__`；不要 `git pull` 后不清除缓存就直接重启。

## 7. systemd 服务配置

服务配置文件路径：`/etc/systemd/system/tradingagents.service`

```ini
[Unit]
Description=TradingAgents Web API
After=network.target

[Service]
Type=simple
User=support
WorkingDirectory=/home/support/TradingAgents
Environment=PATH=/home/support/TradingAgents/venv/bin:/usr/bin:/bin
EnvironmentFile=/home/support/TradingAgents/.env
ExecStart=/home/support/TradingAgents/venv/bin/uvicorn web.app:app --host 0.0.0.0 --port 8000
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

修改服务配置后需重新加载：

```bash
ssh -i $HOME/.ssh/id_rsa_google_longterm support@34.92.13.95 "sudo systemctl daemon-reload && sudo systemctl restart tradingagents"
```
