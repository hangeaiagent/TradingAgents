所有部署操作均在本地通过 Claude Code 完成，Claude Code 使用 SSH 连接服务器执行命令。

**SSH 连接需指定密钥文件**：

```bash
# SSH 连接格式
ssh -i $HOME/.ssh/id_rsa_google_longterm support@34.92.13.95 "命令"

ssh -i $HOME/.ssh/id_rsa_google_longterm support@34.92.13.95 "cd /home/support/TradingAgents && 命令1 && 命令2"
```

## 5. 日常更新

### 5.1 代码更新（Git 拉取）

最常用的更新方式，由 Claude Code 执行。**必须清除 `__pycache__` 缓存**，否则旧字节码可能导致新代码不生效：

```bash
# 标准部署流程（一条命令，不可省略清缓存步骤）
ssh -i $HOME/.ssh/id_rsa_google_longterm support@34.92.13.95 "\
  cd /home/support/TradingAgents && \
  git pull origin main && \
  find . -name '__pycache__' -type d -exec rm -rf {} + 2>/dev/null; \
  sudo systemctl restart tradingagents && \
  sleep 2 && \
  sudo systemctl status tradingagents --no-pager"
```

如有依赖变更，在 `git pull` 之后、重启之前加：

```bash
ssh -i $HOME/.ssh/id_rsa_google_longterm support@34.92.13.95 "cd /home/support/TradingAgents && source venv/bin/activate && pip install -r requirements.txt"
```

> **注意**：`find . -name '__pycache__' -exec rm -rf {} +` 是必须步骤。详见[服务器运维操作 - `__pycache__` 缓存问题](服务器运维操作.md#__pycache__-缓存导致代码未生效重要)。

### 5.2 上传特定文件（SCP）

当需要上传本地文件到服务器时：

```bash
# 上传单个文件
scp -i $HOME/.ssh/id_rsa_google_longterm /Users/a1/work/TradingAgents/文件路径 support@34.92.13.95:/home/support/TradingAgents/文件路径

# 上传目录
scp -i $HOME/.ssh/id_rsa_google_longterm -r /Users/a1/work/TradingAgents/目录 support@34.92.13.95:/home/support/TradingAgents/
```

### 5.3 批量同步（rsync）

同步整个项目（排除不需要的文件）：

```bash
rsync -avz -e "ssh -i $HOME/.ssh/id_rsa_google_longterm" \
  --exclude 'venv' --exclude '.git' --exclude '__pycache__' --exclude '.env' \
  /Users/a1/work/TradingAgents/ support@34.92.13.95:/home/support/TradingAgents/
```

## 8. Web 服务接口

服务启动后可通过 `http://34.92.13.95:8000` 访问：

### 8.1 页面路由

| 端点 | 方法 | 用途 |
|------|------|------|
| `/` | GET | 前端页面（未登录显示登录页，已登录显示分析页） |
| `/sso-callback` | GET | SSO 回调中转页 |
| `/billing` | GET | 用量账单页 |
| `/history` | GET | 分析历史页 |
| `/api/docs` | GET | OpenAPI 文档 |

### 8.2 认证接口（AgentPit OAuth2 SSO）

| 端点 | 方法 | 用途 |
|------|------|------|
| `/api/auth/agentpit/login` | GET | 发起 OAuth2 授权登录 |
| `/api/auth/agentpit/sso` | GET | SSO 静默登录入口（自动单点登录） |
| `/api/auth/agentpit/callback` | GET | OAuth2 回调（处理授权码换 token） |
| `/api/auth/logout` | GET | 退出登录 |
| `/api/auth/login` | GET | 旧路径兼容，跳转到 agentpit/login |
| `/api/auth/callback` | GET | 旧路径兼容，跳转到 agentpit/callback |

**SSO 自动登录流程：**
1. 用户访问首页，前端检查 `sessionStorage.sso_attempted` 标记
2. 如未尝试过 SSO，自动跳转 `/api/auth/agentpit/sso`
3. 后端 302 到 AgentPit OAuth 授权页（state 带 `sso:` 前缀）
4. 授权成功回调后，跳转 `/sso-callback#token=xxx`
5. SSO 回调页清除 URL 中的 token，设置 `sso_completed` 标记，跳转首页
6. 防循环：每个浏览器 session 最多尝试一次 SSO

### 8.3 业务接口

| 端点 | 方法 | 用途 |
|------|------|------|
| `/api/analyze/sync` | POST | 同步股票分析（需登录） |
| `/api/analyze` | POST | SSE 流式分析（需登录） |
| `/api/user/info` | GET | 获取当前用户信息和用量 |
| `/api/history` | GET | 获取分析历史列表 |
| `/api/history/{id}` | GET | 获取单条分析详情 |
| `/skills/{skill_name}.md` | GET | OpenClaw Skill 文件 |

### 8.4 Token 用量上报接口

| 端点 | 方法 | 用途 |
|------|------|------|
| `/api/v1/tokens/report` | POST | Token 消耗上报 |

**请求格式：**
```json
{
  "agent_id": "required",
  "tokens_used": 1000,
  "input_tokens": 600,
  "output_tokens": 400,
  "started_at": "2024-01-01T00:00:00Z",
  "ended_at": "2024-01-01T00:01:00Z",
  "model_name": "gpt-4",
  "request_id": "optional-uuid",
  "metadata": {}
}
```

**认证：** `Authorization: Bearer <api_key_or_access_token>`

**响应：**
```json
{ "success": true, "data": { "id": 1, "agent_id": "...", "tokens_used": 1000, ... } }
```
