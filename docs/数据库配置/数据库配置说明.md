# TradingAgents 数据库配置说明

## 数据库概览

TradingAgents 使用 **PostgreSQL** 作为生产数据库，开发环境可回退到 **SQLite**。

| 环境 | 数据库 | 地址 |
|------|--------|------|
| 生产服务器 (34.92.13.95) | PostgreSQL | 34.150.17.200:5432/agentpit |
| 本地开发 | PostgreSQL（同上）或 SQLite 回退 | 本地 data/trading.db |

## 连接配置

在 `.env` 文件中设置 `DATABASE_URL`：

```bash
DATABASE_URL=postgresql://agentpit:AgentPit%40Sql2026!@34.150.17.200:5432/agentpit
```

**注意事项：**
- 密码 `AgentPit@Sql2026!` 中的 `@` 必须 URL 编码为 `%40`，否则会被解析为主机名分隔符
- 留空 `DATABASE_URL` 则自动使用本地 SQLite（`data/trading.db`）
- PostgreSQL 连接失败时自动回退到 SQLite（容错机制，见 `web/database.py`）

## 表结构

TradingAgents 使用 `trading_` 前缀，与 AgentPit 平台的 `apbase_*` 表共存于同一数据库。

### trading_users — 用户信息

| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | TEXT PK | AgentPit OAuth 用户 ID |
| name | TEXT | 用户名 |
| email | TEXT | 邮箱 |
| avatar | TEXT | 头像 URL |
| created_at | TIMESTAMP | 首次登录时间 |

### trading_analysis_records — 分析记录

| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL PK | 自增 ID |
| user_id | TEXT | 关联用户 |
| ticker | TEXT | 股票代码 |
| trade_date | TEXT | 分析日期 |
| decision | TEXT | BUY/SELL/HOLD |
| tokens_in / tokens_out | INTEGER | Token 用量 |
| llm_calls / tool_calls | INTEGER | 调用次数 |
| elapsed_ms | INTEGER | 耗时（毫秒） |
| model | TEXT | 使用的 LLM 模型 |
| report_json | TEXT | 完整报告 JSON |
| timestamp | TIMESTAMP | 记录时间 |

### trading_token_usage — Token 用量上报

| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL PK | 自增 ID |
| agent_id | TEXT | Agent 标识 |
| bearer_token | TEXT | 认证 Token |
| tokens_used | INTEGER | 总 Token |
| input_tokens / output_tokens | INTEGER | 输入/输出 Token |
| model_name | TEXT | 模型名称 |
| request_id | TEXT | 请求 ID |
| metadata_json | TEXT | 元数据 JSON |
| started_at / ended_at | TEXT | 起止时间 |
| created_at | TIMESTAMP | 记录时间 |

## 代码位置

- **数据库模块**: `web/database.py` — 连接池管理、表创建、CRUD 操作、SQLite 回退
- **环境变量读取**: `web/database.py` 第 22 行 `DATABASE_URL = os.environ.get("DATABASE_URL", "")`
- **连接池**: psycopg2 `ThreadedConnectionPool`，min=2, max=10

## 迁移机制

首次连接 PostgreSQL 时，代码自动执行：
1. 创建 `trading_*` 表（`CREATE TABLE IF NOT EXISTS`）
2. 检查本地 `data/trading.db` 和 `data/web_usage.json`，将旧数据迁移到 PostgreSQL
3. 迁移完成后将 SQLite 文件重命名为 `.bak`
