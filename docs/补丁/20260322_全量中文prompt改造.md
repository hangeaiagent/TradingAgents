# 补丁：全量中文 Prompt 改造

**日期**：2026-03-22
**提交**：`5c0e2c7` + `e8d18b2`
**范围**：Agent 层（12 个 agent + language 工具模块）

---

## 一、问题背景

前一版（`0a4e867`）的中文输出方案采用 **末尾追加 `lang_suffix`** 的方式：在英文 system prompt 结尾附加一段中文指令。实际效果不佳：

- 整个 system prompt 主体为大段英文，LLM（特别是 Gemini Flash）倾向于沿用英文回复
- 工具调用阶段（"I'll start by getting..."）完全用英文描述
- 交易员（Trader）输出仍为英文
- 中文指令位于 prompt 末尾，权重太低，LLM 注意力不足

## 二、解决方案

### 核心改动

1. **整个 system prompt 翻译为中文** — 当 `output_language="zh"` 时，所有 agent 使用完整的中文 prompt，而非在英文 prompt 后追加一句
2. **中文指令放在 prompt 最前面** — `【核心规则】你必须始终使用简体中文输出所有内容。严禁使用英文描述你的分析过程或结论。` 作为 system prompt 第一句
3. **`language.py` 重写** — 移除旧的 `LANG_SUFFIX` 追加机制，改为 `is_chinese()` 判断 + 完整中英文双语 boilerplate

### 旧方案 vs 新方案

| 项目 | 旧方案（0a4e867） | 新方案（5c0e2c7 + e8d18b2） |
|------|-------------------|----------------------------|
| prompt 主体语言 | 英文 | 中文（zh 模式下） |
| 中文指令位置 | system prompt 末尾 | system prompt 最前面 |
| 指令强度 | "请用中文" | "【核心规则】严禁使用英文" |
| 工具调用描述 | 英文 | 中文 |
| 覆盖范围 | 仅报告正文 | 全流程（工具调用描述 + 报告正文） |

## 三、修改文件清单

### `tradingagents/agents/utils/language.py`

**重写**，移除旧的追加式 `LANG_SUFFIX`，新增：

- `is_chinese(config)` — 判断是否中文模式
- `ANALYST_BOILERPLATE_ZH` — 4 个 analyst 共用的中文协作框架 prompt
- `ANALYST_BOILERPLATE_EN` — 英文版（保持向后兼容）
- `get_analyst_boilerplate(config)` — 根据语言返回对应 boilerplate

**中文 boilerplate 首行**：
```
【核心规则】你必须始终使用简体中文输出所有内容，包括思考过程、工具调用说明和最终报告。
仅保留股票代码（如 AAPL、600588.SS）、技术指标名称（如 RSI、MACD）和数值为英文原文。
严禁使用英文描述你的分析过程或计划。
```

### 4 个分析师 Agent

每个文件的 `system_message`（角色指令）和外层 `boilerplate`（协作框架）均改为中英文双版本：

| 文件 | 中文角色 |
|------|---------|
| `analysts/market_analyst.py` | 市场分析师 — 技术指标选择与趋势分析 |
| `analysts/fundamentals_analyst.py` | 基本面分析师 — 财务报表与公司分析 |
| `analysts/news_analyst.py` | 新闻分析师 — 宏观新闻与趋势 |
| `analysts/social_media_analyst.py` | 舆情分析师 — 社交媒体情绪分析 |

**改动模式**（以 `market_analyst.py` 为例）：
```python
# 旧：英文 prompt + 末尾追加
system_message = "You are a trading assistant..."
prompt_template = "..." + "{lang_suffix}"

# 新：根据语言选择完整 prompt
if zh:
    system_message = "你是一位交易助理，负责分析金融市场..."
else:
    system_message = "You are a trading assistant..."
boilerplate = get_analyst_boilerplate(config)  # 中文或英文
```

## 四、技术细节

### Prompt 结构变化（Analyst 类）

```
旧结构：
┌─────────────────────────────────────┐
│ You are a helpful AI assistant...   │ ← 英文协作框架
│ {system_message}                    │ ← 英文角色指令
│ current_date, ticker                │
│ {lang_suffix}                       │ ← 末尾追加的中文指令（被忽视）
└─────────────────────────────────────┘

新结构：
┌─────────────────────────────────────┐
│ 【核心规则】严禁使用英文...          │ ← 最前面，最高优先级
│ 你是一个有用的 AI 助手...            │ ← 中文协作框架
│ 当前日期，分析公司                   │
│ {system_message}                    │ ← 中文角色指令
└─────────────────────────────────────┘
```

### Prompt 结构变化（非 Analyst 类）

```
旧结构：
f"""You are a Bull Analyst...""" + lang_suffix

新结构：
if zh:
    f"""【核心规则】严禁使用英文...
    你是一位多头分析师..."""
else:
    f"""You are a Bull Analyst..."""
```

### 修复的 Bug

- **`{system_message}` 重复注入** — boilerplate 和 agent 代码中各有一个 `{system_message}` 占位符，导致角色指令在最终 prompt 中出现两次。修复后 boilerplate 中移除 `{system_message}`，仅在 agent 代码中注入一次。

## 五、配置说明

无需额外配置。Web 端默认 `output_language="zh"`（由前端 `localStorage` 的 `ta_lang` 控制），中文 prompt 自动生效。

如需切换回英文，在前端点击 EN 按钮或设置 `config["output_language"] = None`。
