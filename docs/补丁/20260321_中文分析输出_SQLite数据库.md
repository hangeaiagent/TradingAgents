# 补丁：中文分析输出 + SQLite 数据库存储

**日期**：2026-03-21
**范围**：后端 + 前端 + Agent 层

---

## 一、变更概述

本次补丁包含两个核心功能：

1. **SQLite 数据库替代 JSON 文件存储** — 用量数据、分析记录、用户信息全部持久化到 SQLite，支持并发读写、月度聚合查询
2. **分析报告中文输出** — 12 个 Agent 的 prompt 末尾注入中文指令后缀，Web 端默认以中文输出分析报告

---

## 二、新增文件

### `web/database.py`

SQLite 数据库封装模块。

- **数据库文件**：`data/trading.db`
- **表名前缀**：`trading_`（与 agentpit 完全隔离）
- **表结构**：
  - `trading_users` — 用户信息（user_id, name, email, avatar）
  - `trading_analysis_records` — 分析记录（含 report_json 存完整报告 JSON）
  - 索引：`idx_trading_records_user(user_id, timestamp)`
- **线程安全**：`threading.local()` 每线程独立 connection
- **WAL 模式**：支持并发读写
- **自动迁移**：启动时检测 `data/web_usage.json`，导入后重命名为 `.json.bak`

**核心方法**：

| 方法 | 说明 |
|------|------|
| `upsert_user(user_id, name, email, avatar)` | 创建或更新用户，非空字段覆盖 |
| `record_analysis(user_id, ticker, ..., report)` | 插入分析记录，返回含 `id` 的 dict |
| `get_history(user_id)` | 返回用户历史列表（不含 report_json），按时间倒序 |
| `get_record(record_id, user_id)` | 返回单条记录含解析后的 `report` dict |
| `get_usage_summary(user_id)` | SQL 聚合：总计 + 月度分组 + 最近 50 条 |

## 四、API 变更

### `POST /api/analyze`

**请求体新增字段**：

```json
{
  "language": "zh"  // 可选，默认 "zh"；设为 null 或 "en" 则英文输出
}
```

### `GET /api/history`

**响应变更**：列表项中 `index` 字段改为 `id`（数据库自增主键）

```json
{
  "items": [
    {
      "id": 1,           // 原为 "index": 0
      "ticker": "AAPL",
      "has_report": 1,   // SQLite 返回 1/0 而非 true/false
      ...
    }
  ]
}
```

### `GET /api/history/{record_id}`

路径参数从 list index 改为数据库 record ID。

## 五、数据迁移

启动时自动执行：

1. 检测 `data/web_usage.json` 是否存在
2. 若存在，逐条插入 `trading_analysis_records` 表
3. 迁移完成后将 JSON 文件重命名为 `web_usage.json.bak`
4. 后续启动不再重复迁移

## 六、配置说明

### 中文输出控制

```python
# tradingagents/default_config.py
"output_language": None,   # None = 英文（默认），"zh" = 中文

# Web 层默认行为（web/app.py build_config）
config["output_language"] = req.language or "zh"  # Web 端默认中文
```

- CLI 使用时默认英文（`None`），需手动设置 `config["output_language"] = "zh"`
- Web 端根据前端 i18n 语言切换自动传入
