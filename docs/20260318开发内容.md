# 最新提交分析

## 1. `b80df0c` — OpenClaw Skill 支持 + 无头 CLI 入口

这是核心功能提交，新增了 **OpenClaw 技能平台集成** 和 **独立脚本运行能力**：

**新增文件：**

| 文件 | 作用 |
|------|------|
| `scripts/analyze.py` | **核心** — 无头 CLI 入口，支持 `--ticker`、`--tickers`（批量）、`--depth`（full/quick）、`--llm`、`--model`、`--base-url` 参数 |
| `scripts/run.sh` | Shell 包装器，自动创建 venv 并调用 `analyze.py` |
| `scripts/install.sh` | 依赖安装脚本，创建 venv + `pip install -e .` |
| `SKILL.md` | OpenClaw 技能清单，定义了技能名称、描述、命令格式、使用示例 |
| `CLAUDE.md` | Claude Code 项目指引文件 |
| 两个中文 One-API 集成文档 | One-API 代理配置方案 |

**关键设计点：**
- `analyze.py` 支持 `ONE_API_KEY` → `OPENAI_API_KEY` 自动降级，兼容 One-API 代理
- `ONE_API_BASE_URL` / `ONE_API_GEMINI_MODEL` 环境变量支持自定义 LLM 端点
- 5 秒内必须有输出（OpenClaw 超时要求），所以先 `print` 再加载模块
- `format_report()` 将 LangGraph 最终状态格式化为 Markdown 报告

## 2. `d63b4a0` — 文档 + OpenClaw Skills + Web 模块

**新增文件：**

| 文件 | 作用 |
|------|------|
| `web/app.py` | **FastAPI Web 后端**，提供 `/api/analyze`（SSE 流式）和 `/api/analyze/sync`（同步）两个接口 |
| `web/static/index.html` | **单页面前端**，暗色主题的股票分析 Web UI，支持输入 ticker/日期/深度/LLM 提供商 |
| `skills_openclaw/tradingagents-demo.md` | OpenClaw 演示版技能定义 |
| `skills_openclaw/tradingagents-pro.md` | OpenClaw 专业版技能定义（需 AgentPit 认证） |
| `docs/skill/session-registration-spec.md` | Session 注册技术规范（测试/正式用户） |
| `docs/skill/werewolf*.md` | 狼人杀游戏相关技能文档 |
| `docs/skill/部署skill.md` | Skill 部署指南 |

**Web 模块关键特性：**
- `POST /api/analyze` — SSE 流式返回分析进度和结果
- `POST /api/analyze/sync` — 同步返回完整 JSON（供 curl/OpenClaw 调用）
- `GET /skills/{name}.md` — 动态提供 skill 文件供 OpenClaw 安装
- 前端用原生 JS + Fetch API 消费 SSE，无框架依赖
- 分析结果按 BUY/SELL/HOLD 分色显示，报告按可折叠章节展示

---

## 总结

这两次提交的核心目标是将 TradingAgents 从一个纯 CLI/Python 库扩展为：
1. **OpenClaw 平台可调用的 Skill**（通过 `SKILL.md` + `scripts/analyze.py`）
2. **独立 Web 服务**（通过 `web/app.py` + FastAPI + 前端页面）
3. **支持 One-API 代理**，可通过统一网关调用多种 LLM
